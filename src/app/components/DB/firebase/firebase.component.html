De manera simple "FIREBASE", aloja nuestros datos y nos presta el servicio de backend para solo enviar las peticiones directo a la ruta de los servicios.

Anteriormente lo que consideraba de "FIREBASE", era que sus servicios eran de una base de datos no relacional, a la cual conectaba mi aplicación de "ANGULAR" y podia realizar todas las operaciones CRUD, ya.

En realidad son muchos servicios para aplicaciones móviles y web de los cuales son asequibles gratuitamente con limitantes que no evitan desarrollar en pruebas cualquier aplicación.

Actualmente se requiere tener muy en cuenta el control de acceso y para conectar una base de datos "FIREBASE" a una aplicación de angular.

Actualmente se puede hacer en modo de prueba por pocos días, luego "FIREBASE" nos recordará por medio de email que se deben configurar las reglas de acceso, ya que cualquier usuario puede acceder a la base de datos y realizar cambios sin autorización. Luego de eso se deshabilita el acceso a la base de datos y se vuelve tedioso estar habilitando el acceso por pocos días.

Lo ideal sería aplicar el proceso adecuado de control de acceso a los datos mediante un login de autorización de usuario y luego de eso permitir el acceso a los datos mediante un token.

REALTIME DATABASE

(SECURE RULES) REGLAS SEGURAS EN ANGULAR

Una manera efectiva de aplicar reglas seguras se puede lograr utilizando el "FIREBASE LOCAL EMULATOR SUITE", que es un conjunto de herramientas para probar localmente como funciona todo antes de aplicar cambios directamente en "FIREBASE".
