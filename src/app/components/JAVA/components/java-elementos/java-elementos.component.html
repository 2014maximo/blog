ESTRUCTURA DE ARCHIVOS

Los proyectos de Java presentan numerosas estructuras de archivos que varían según el editor de texto, el tipo de arquitectura o el enfoque de aplicativo. Miremos algunas y para que todo eso.

Básico de IntellijIdea
Model View Controller

RESOURCES: Aquí se almacenan los archivos de recursos que tu aplicación pueda necesitar, como archivos de configuración, archivos de propiedades, imágenes, etc.
TEST: Este directorio puede contener las pruebas unitarias o de integración para tu proyecto. Por lo general, se organiza de manera similar al directorio "src", pero específicamente para pruebas.
BIN: Este directorio puede contener los archivos binarios generados durante la compilación, como archivos de clase (.class) y archivos JAR ejecutables.
LIB: Aquí es donde se almacenan las bibliotecas externas (JARs) que tu proyecto pueda necesitar. Pueden ser bibliotecas de terceros o tus propias bibliotecas compartidas entre varios proyectos.
SRC: Este directorio contiene todo el código fuente de tu proyecto Java. Por lo general, se organiza en paquetes que reflejan la estructura de tus clases.
